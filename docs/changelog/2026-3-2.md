# OpenClaw 2026.3.2 版本解读

> 发布时间：2026年3月2日  
> 官方发布：[GitHub Release](https://github.com/openclaw/openclaw/releases/tag/v2026.3.2)

## 🎯 核心亮点

### 1. 外部密钥管理

完整的 `openclaw secrets` 工作流：

```bash
openclaw secrets audit      # 审计密钥使用
openclaw secrets configure  # 配置密钥引用
openclaw secrets apply      # 应用密钥更改
openclaw secrets reload     # 重新加载密钥
```

- 支持 64 个凭证目标的 SecretRef
- 运行时快照激活
- 严格的目标路径验证
- 更安全的迁移清理

### 2. PDF 分析工具

新增一流的 `pdf` 工具：

- 原生 Anthropic 和 Google PDF 支持
- 非原生模型的提取回退
- 可配置默认值：
  ```json
  {
    "agents": {
      "defaults": {
        "pdfModel": "claude-3-5-sonnet-20241022",
        "pdfMaxBytesMb": 32,
        "pdfMaxPages": 100
      }
    }
  }
  ```

### 3. ACP 线程绑定 Agent

ACP agent 成为线程会话的一流运行时：

- `acp` spawn/send 调度集成
- acpx 后端桥接
- 生命周期控制
- 启动协调和运行时清理

## 🔧 重要改进

### Sessions 附件支持

`sessions_spawn` 支持内联文件附件：

```javascript
sessions_spawn({
  attachments: [
    {
      filename: "data.json",
      content: "base64-encoded-content",
      encoding: "base64"
    }
  ]
})
```

### Telegram 流式默认值

- 默认 `streaming: "partial"`
- 新设置自动获得实时预览流
- DM 使用 `sendMessageDraft`

### 多语言支持

- Web UI 新增德语（`de`）支持
- 自动渲染语言选项

### 工具增强

**Diffs 工具**

- PDF 文件输出支持
- 渲染质量自定义控制
- 推荐用于压缩图像的频道

**Memory Ollama 嵌入**

```json
{
  "memorySearch": {
    "provider": "ollama",
    "fallback": "ollama"
  }
}
```

## 🐛 重要修复

### 插件系统

- 验证和规范化插件命令
- 防止格式错误的命令规范崩溃
- Telegram 原生菜单规范化保护

### 平台修复

**Discord**
- 捕获启动时 gateway 错误事件
- 早期 `Fatal Gateway error: 4014` 处理

**Feishu**
- 多应用提及路由保护
- 会话内存钩子对齐
- 群组系统提示转发

**Slack**
- 提及格式化加固
- 空文本负载保护

### 安全加固

- Gateway/Subagent TLS 配对
- 本地 gateway-client 后端自连接
- Docker/LAN 设置恢复

## 🚀 升级建议

### 必读变更

**Onboarding 默认工具配置**

新本地安装默认 `tools.profile` 为 `messaging`：

```json
{
  "tools": {
    "profile": "messaging"  // 新默认值
  }
}
```

如需编码工具，显式配置为 `coding`。

**ACP 调度默认启用**

```json
{
  "acp": {
    "dispatch": {
      "enabled": true  // 新默认值
    }
  }
}
```

如需暂停，设置为 `false`。

### 推荐配置

**外部密钥管理**

```bash
# 2. 配置密钥引用

```
