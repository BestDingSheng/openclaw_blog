## 背景：AI Agent 上下文管理的痛点

### 传统方案的局限性

在 ContextEngine 出现之前，AI Agent 的上下文管理面临几个核心问题：

1. **硬编码策略**：上下文压缩和管理逻辑深度嵌入在 Agent 核心代码中
2. **单一策略**：只能使用滑动窗口压缩，无法适应不同场景需求
3. **会话隔离**：关闭会话后上下文丢失，每次对话都从零开始
4. **细节丢失**：摘要压缩会丢失重要细节，影响长期对话质量

### 开发者的困扰

```javascript
// 传统方式：修改核心代码才能自定义上下文策略
// 风险高、维护难、无法复用
class LegacyAgent {
  async processMessage(message) {
    // 硬编码的上下文管理逻辑
    if (this.context.length > MAX_TOKENS) {
      this.context = this.summarizeOldMessages(this.context);
    }
    // ...
  }
}
```

## ContextEngine：插件化的革命性解决方案

### 核心设计理念

ContextEngine 将整个上下文生命周期抽象为一个标准接口，开发者可以通过插件的方式实现自定义的上下文管理策略，而无需修改 OpenClaw 的核心代码。

### 七大生命周期钩子

ContextEngine 提供了七个关键的生命周期钩子：

```typescript
interface ContextEngine {
  // 1. 引擎启动初始化
  bootstrap(): Promise<void>;
  
  // 2. 消息摄取处理
  ingest(message: Message): Promise<void>;
  
  // 3. 上下文组装（核心）
  assemble(budget: TokenBudget): Promise<AssembledContext>;
  
  // 4. 上下文压缩
  compact(): Promise<void>;
  
  // 5. 对话轮次后处理
  afterTurn(result: TurnResult): Promise<void>;
  
  // 6. 子 Agent 准备
  prepareSubagentSpawn(config: SubagentConfig): Promise<void>;
  
  // 7. 引擎清理
  cleanup(): Promise<void>;
}
```

### 3. RAG 增强的 ContextEngine 实现

```typescript
class RAGContextEngine implements ContextEngine {
  private vectorStore: VectorStore;
  private sessionGraph: ConversationDAG;
  
  async bootstrap(): Promise<void> {
    // 连接向量数据库
    this.vectorStore = await connectToVectorDB(this.config.dbUrl);
    this.sessionGraph = new ConversationDAG();
    
    console.log('RAG ContextEngine initialized');
  }
  
  async ingest(message: Message): Promise<void> {
    // 将消息添加到对话图
    const node = this.sessionGraph.addNode(message);
    
    // 生成嵌入并存储到向量数据库
    const embedding = await this.generateEmbedding(message.content);
    await this.vectorStore.upsert(node.id, embedding, {
      content: message.content,
      timestamp: message.timestamp,
      role: message.role,
      metadata: message.metadata
    });
  }
  
  async assemble(budget: TokenBudget): Promise<AssembledContext> {
    const currentQuery = this.getCurrentQuery();
    
    // 基于当前查询检索相关历史
    const relevantHistory = await this.vectorStore.query(
      await this.generateEmbedding(currentQuery),
      {
        topK: this.config.maxRetrievalDocs,
        threshold: this.config.similarityThreshold
      }
    );
    
    // 构建上下文
    const context = this.buildContextFromRetrieval(
      relevantHistory, 
      budget
    );
    
    return {
      messages: context.messages,
      systemPrompt: context.systemPrompt,
      tokenCount: context.tokenCount,
      metadata: {
        retrievalCount: relevantHistory.length,
        strategy: 'rag-enhanced'
      }
    };
  }
  
  async compact(): Promise<void> {
    // 清理过期的向量数据
    const cutoffTime = Date.now() - (7 * 24 * 60 * 60 * 1000); // 7天前
    await this.vectorStore.deleteByTimestamp(cutoffTime);
  }
}
```

### 4. 高级场景：分支对话管理

```typescript
class BranchingContextEngine implements ContextEngine {
  private conversationTree: ConversationTree;
  
  async assemble(budget: TokenBudget): Promise<AssembledContext> {
    const currentBranch = this.conversationTree.getCurrentBranch();
    const parentContext = this.conversationTree.getParentContext();
    
    // 智能合并分支上下文
    const mergedContext = this.mergeBranchContexts(
      currentBranch, 
      parentContext, 
      budget
    );
    
    return {
      messages: mergedContext,
      branchInfo: {
        branchId: currentBranch.id,
        parentBranch: parentContext?.id,
        depth: currentBranch.depth
      }
    };
  }
  
  async prepareSubagentSpawn(config: SubagentConfig): Promise<void> {
    // 为子 Agent 创建独立的上下文分支
    const newBranch = this.conversationTree.createBranch({
      parentId: this.conversationTree.getCurrentBranch().id,
      purpose: config.purpose,
      isolationLevel: config.isolationLevel
    });
    
    config.contextBranch = newBranch.id;
  }
}
```

## 性能优化与最佳实践

### 1. 异步处理优化

```typescript
class OptimizedContextEngine implements ContextEngine {
  private processingQueue = new AsyncQueue();
  
  async ingest(message: Message): Promise<void> {
    // 异步处理，不阻塞主流程
    this.processingQueue.add(async () => {
      await this.processMessageEmbedding(message);
    });
  }
  
  async assemble(budget: TokenBudget): Promise<AssembledContext> {
    // 并行检索和处理
    const [vectorResults, graphResults] = await Promise.all([
      this.vectorStore.query(this.currentQuery),
      this.conversationGraph.findRelevantPaths()
    ]);
    
    return this.mergeResults(vectorResults, graphResults, budget);
  }
}
```

### 2. 缓存策略

```typescript
class CachedContextEngine implements ContextEngine {
  private contextCache = new LRUCache<string, AssembledContext>(100);
  
  async assemble(budget: TokenBudget): Promise<AssembledContext> {
    const cacheKey = this.generateCacheKey(budget);
    
    if (this.contextCache.has(cacheKey)) {
      return this.contextCache.get(cacheKey)!;
    }
    
    const context = await this.buildContext(budget);
    this.contextCache.set(cacheKey, context);
    
    return context;
  }
}
```

class EnterpriseContextEngine implements ContextEngine {
  async bootstrap(): Promise<void> {
    // 多租户隔离
    this.tenantManager = new TenantManager();
    // 合规性检查
    this.complianceChecker = new ComplianceChecker();
