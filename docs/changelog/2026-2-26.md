# OpenClaw 2026.2.26 版本解读

> 发布时间：2026年2月26日  
> 官方发布：[GitHub Release](https://github.com/openclaw/openclaw/releases/tag/v2026.2.26)

## 🎯 核心亮点

### 1. 外部密钥管理（Highlight）

完整的密钥管理工作流：

```bash
openclaw secrets audit      # 审计密钥使用
openclaw secrets configure  # 配置密钥引用
openclaw secrets apply      # 应用密钥更改
openclaw secrets reload     # 重新加载密钥
```

**特性：**
- 运行时快照激活
- 严格的 `secrets apply` 目标路径验证
- 更安全的迁移清理
- 仅引用的认证配置文件支持

### 2. ACP 线程绑定 Agent

ACP agent 成为线程会话的一流运行时：

- `acp` spawn/send 调度集成
- acpx 后端桥接
- 生命周期控制
- 启动协调
- 运行时清理
- 合并的线程回复

### 3. Agent 路由 CLI

新增 agent 绑定管理命令：

```bash
openclaw agents bindings    # 列出绑定
openclaw agents bind        # 创建绑定
openclaw agents unbind      # 删除绑定
```

- 账户范围的路由管理
- 频道到账户绑定升级
- 角色感知绑定身份处理

## 🔧 重要改进

### Codex WebSocket 传输

- 默认 WebSocket 优先（`transport: "auto"`）
- SSE 回退支持
- 保持显式传输覆盖

### Onboarding 插件

频道插件可以拥有交互式 onboarding 流程：

- 可选的 `configureInteractive` 钩子
- `configureWhenConfigured` 钩子
- 保留通用回退路径

### Android 节点

**设备能力**

```bash
nodes device_status    # 设备状态
nodes device_info      # 设备信息
```

**通知列表**

```bash
nodes notifications_list    # 列出活动通知
```

### 认证和安全

**Gemini CLI OAuth 警告**

- 显式账户风险警告
- 启动前确认门控
- 文档中的注意事项

## 🐛 重要修复

### Telegram DM Allowlist

- 强制执行 `dmPolicy: "allowlist"` 要求
- 使用有效的账户加父级配置
- 对齐 `openclaw doctor` 检查
- 防止升级后静默丢弃 DM

### 交付队列恢复

- 防止重试饥饿
- 持久化 `lastAttemptAt`
- 延迟恢复重试直到退避窗口

### 平台修复

**Google Chat**
- 保持 `startAccount` 挂起直到中止
- 防止 webhook 模式下的自动重启循环

**Nextcloud Talk**
- 保持 `startAccount` 挂起
- 关闭时停止 webhook 监视器
- 防止 `EADDRINUSE` 重启循环

**Microsoft Teams**
- 立即确认 `fileConsent/invoke`
- 防止虚假超时横幅

### 临时目录权限

- 创建后强制 `0700` 权限
- 自我修复现有可写临时目录
- `umask 0002` 安装兼容性

### 队列和 Cron 可靠性

- 加固通道排空
- 保证 `draining` 标志重置
- 在重启排空窗口拒绝新入队
- 添加 `/stop` 队列积压截止

### 打字指示器

- 始终在 `agent-runner` 完成时标记调度空闲
- 添加最大持续时间保护网
- 统一跨频道抑制
- 防止卡住的打字指示器

## 🚀 升级建议

### 推荐配置

**外部密钥管理**

```bash
# 1. 审计当前密钥
openclaw secrets audit

# 2. 配置引用
openclaw secrets configure

# 3. 应用更改
openclaw secrets apply
```

**ACP 线程绑定**

```json
{
  "acp": {
    "enabled": true,
    "backend": "acpx",
    "dispatch": {
      "enabled": true
    }
  }
}
```

**Agent 路由**

```bash
# 查看当前绑定
openclaw agents bindings

# 创建新绑定
openclaw agents bind --channel telegram --account my-bot --agent my-agent
```
